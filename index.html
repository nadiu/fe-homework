Основні команди

git config --global user.name - встановлення імені користувача

git config --global user.email - встановлення адреси електронної пошти

git init - створює порожній репозиторій git

git status - для перегляду статусу файлів в яких були зміни

git add - додає всі файли

git commit -збереження зміни

git log - показує історію сомітів які були створені

git checkout - для переключення на існуючі вітку

git branch - для створення нової вітки

git push - для відправлення вітки на сервер

git pull - для отримання даних з серверу

git merge - для об'єднання декількох віток в одну

git clone - для створення копії існуючого git репозиторія

.gitignore - для приховування файлів і папок від git

Advanced

git revert - безпечний спосіб скасувати операцію без втрати історії коммітів. Команда скасовує дії минулих коммітів, створюючи новий, що містить всі скасовані зміни.

git reset - видаляє коміт, за допомогою перестановки HEAD на коміт, який ви вкажете - попередні зміни скасовуються залежно від режиму. Є три режими: soft reset, hard resed, mixed.

git rebase - переміщує історію всієї вітки поверх іншої, переписуючи історію проекту новими комітами.

git cherry-pick - бере зміни, що вносяться одним коммітом, і намагається повторно застосувати їх у вигляді нового комміту в поточній вітці.

git stash - призначена для того, щоб помістити поточні зміни, які ви здійснили у файлах, в окреме сховище, та повернути файли до вихідного стану.
